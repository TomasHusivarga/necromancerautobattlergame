# Roadmap

This file tracks what to do next. Keep it practical and current.

## Now

- Keep project state visible through `PROJECT_DECISIONS.md`, `ASSET_STATUS.md`, `ROADMAP.md`, and `PROMPT_LIBRARY.md`.
- Generate Battle V5: same approved style as V4, but fewer individual units, simpler terrain, and clearer squad formation shapes.
- Define how lab-built zombies map into simplified battle visuals.
- Use `PART_STAT_TAG_SYSTEM.md` to write the first MVP part list and data examples.

## Next

- Write `LAB_UI_LAYOUT_SPEC.md` based on Lab V4.
- Write `BATTLE_UI_LAYOUT_SPEC.md` after Battle V5.
- Write `BATTLE_RIG_V1.md` for simplified high-angle battle units.
- Define the first 18 MVP parts using the stat/tag system.

## Later

- Create first production lab zombie part set.
- Create first battle zombie rig test.
- Create first terrain patch, UI kit, and spell/VFX set.
- Build Unity prototype with placeholder art first.
- Add animation, balancing, rewards, and progression after the loop works.

## Not MVP

- Multiplayer.
- Rankings, matchmaking, accounts, servers.
- Large maps.
- Advanced procedural content.
- Complex RTS micro-control.
- Large production asset batches before the pipeline is proven.
